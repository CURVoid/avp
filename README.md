# Usage
Avp `Result` type currently implements such rust idiomatic methods as:
- `unwrap`
- `unwrap_err`, this one is not exactly from rust but thats only the way you can extract error from result
- `is_ok`
- `is_err`
- `map`
- `map_err`
- `or_else`
- `and_then`
